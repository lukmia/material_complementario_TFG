En este documento, se proporcionan respuestas a las preguntas frecuentes sobre la migración de las APIs de Google Fit a las APIs de Android Health.

¿A quiénes afecta esta migración?

Todos los desarrolladores que usan las APIs de Google Fit y las APIs de REST de Google Fit

¿Qué cambiará?

  • Las APIs de Google Fit, incluida la API de REST de Google Fit, dejarán de estar disponibles en 2026.
  • A partir del 1 de mayo de 2024, los desarrolladores no podrán registrarse para usar estas APIs.

¿Cómo empiezo?

Para proporcionar una transición fluida a tus usuarios, consulta la guía de migración.

¿Qué puedo hacer con mis APIs de Google Fit durante el período de migración?

Si eres desarrollador y usas las APIs de Google Fit para Android, te recomendamos que migres a las APIs de Android Health ahora para garantizar un servicio ininterrumpido para tus usuarios. Las APIs de Android Health ofrecen varias ventajas en comparación con las APIs de Fit, incluidas las siguientes:

  • Registro de pasos: Android Health proporcionará la API de Recording para dispositivos móviles, que no requiere una cuenta de usuario de Google ni que solicites acceso a los alcances de la API. También es una forma más eficiente de recuperar datos de recuento de pasos en términos de consumo de batería. La API de Recording para dispositivos móviles se lanzó con el registro de pasos.
  • Health Connect: Si realizas la integración con Health Connect, tu app para Android podrá acceder a los datos de un ecosistema cada vez más grande de apps con una sola conexión. Además, los datos se almacenan integrados en el dispositivo, lo que permite que el usuario tenga el control total de sus datos.

¿Qué sucederá con las APIs de Google Fit después del período de migración?

Las APIs de Google Fit ya no estarán disponibles para su uso. Después del período de migración, ya deberías haber migrado a Health Connect o a otras APIs de Android Health.

¿Qué sucederá con las APIs de REST de Google Fit después del período de migración?

No hay una alternativa a la API de REST de Fit.

Recomendamos a los usuarios de la API de REST de Fit que migren a las APIs de Android Health. Consulta cuál se adapta mejor a tus necesidades y a las de tus usuarios:

  • Health Connect: Health Connect unifica los datos de la cartera de dispositivos y apps de Android en un ecosistema, lo que proporciona una plataforma de salud común para los desarrolladores de Android. Health Connect proporciona una API para leer y escribir los datos de salud y actividad física del usuario, estandariza el esquema de datos durante el almacenamiento y centraliza el control de permisos.
  • API de Fitbit Web: La API de Fitbit Web es una interfaz independiente de la plataforma que se usa para integrar el ecosistema de Fitbit. Brinda a los usuarios la capacidad de almacenar, compartir y administrar sus datos directamente en la nube. En la API de Fitbit Web, los datos del usuario se vinculan a su cuenta de Fitbit en lugar de hacerlo a su dispositivo. Esto significa que esta API se centra en la cuenta y no en el dispositivo.

¿Cómo puedo obtener ayuda?

Si encuentras un error o necesitas ayuda con la migración, envía tus comentarios a través de nuestro seguimiento de problemas.
